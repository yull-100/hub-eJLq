# 狼人杀多智能体博弈系统 - 前端

基于 Vue 3 的狼人杀博弈系统前端界面，通过 HTTP API 与后端游戏引擎交互。

## 技术栈

- **Vue 3**（CDN 引入，无需构建工具）
- 原生 CSS（深色游戏主题）
- 原生 JavaScript（ES Module）

## 目录结构

```
fronted/
├── index.html          # 主页面（Vue 3 CDN + 组件模板）
├── css/
│   └── style.css       # 全局样式
└── js/
    └── app.js          # Vue 应用逻辑（API 调用 + 状态管理）
```

## 启动方式

前端页面由后端统一提供，无需单独启动：

```bash

# 安装依赖
pip install -r ../requirements.txt

# 启动服务（前端自动可访问）
python ../main.py --server

# 浏览器打开 http://localhost:8000
```

## 页面功能

### 游戏创建
- 配置玩家数量（3-12 人）和最大轮数（1-50 轮）
- 点击「创建游戏」调用后端初始化角色分配

### 游戏控制
- **单步执行**：按 夜晚→白天讨论→白天投票 逐阶段推进
- **完整运行**：直接执行整局游戏至结束
- 实时显示当前轮次、阶段、存活人数

### 玩家列表
- 显示所有玩家存活状态（🟢 存活 / 💀 死亡）
- 狼人阵营玩家红色边框高亮
- 游戏结束后显示每位玩家的真实身份和阵营

### 消息面板
- **消息 Tab**：显示游戏中所有公共/系统消息
- **日志 Tab**：完整游戏日志（含内心独白、投票记录）
- **步骤结果 Tab**：显示接口返回的原始 JSON 数据

### 游戏结束
- 弹出遮罩层展示获胜阵营
- 列出所有玩家的最终身份和阵营
- 点击「返回」重新开始新游戏

## API 接口依赖

| 接口 | 用途 |
|------|------|
| `POST /api/game` | 创建游戏 |
| `GET /api/game/{id}` | 获取状态和玩家信息 |
| `POST /api/game/{id}/step` | 单步执行 |
| `POST /api/game/{id}/run` | 完整运行 |
| `GET /api/game/{id}/messages` | 获取消息 |
| `GET /api/game/{id}/logs` | 获取日志 |

## 开发说明

- 所有 API 请求走相对路径 `/api/...`，由后端同源代理
- Vue 3 通过 CDN 引入（`unpkg.com/vue@3`），无需 `npm install`
- 页面样式采用 CSS 变量，主题色在 `:root` 中定义，方便自定义
- 后端重新加载后，前端页面按 F5 刷新即可
